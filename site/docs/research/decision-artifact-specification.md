# Decision Artifact Specification

A Decision Artifact is the core data structure defined by the Decision Integrity Protocol (DIP).

It represents a complete record of a decision event that can be independently verified.

## Structure

Example decision artifact:

```json
{
  "decision_id": "dec-2026-001",
  "inputs": {
    "customer_id": "CUST-12345",
    "risk_score": 0.82
  },
  "outputs": {
    "decision": "approve"
  },
  "metadata": {
    "system": "fraud-detection-service",
    "model_version": "v3.2"
  },
  "signature": {
    "algorithm": "ed25519",
    "public_key": "PUBLIC_KEY",
    "value": "SIGNATURE"
  },
  "timestamp": "2026-03-05T12:00:00Z",
  "version": "1.0"
}
Fields
decision_id

Unique identifier for the decision.

inputs

Inputs used by the decision system.

outputs

Result produced by the system.

metadata

Context describing the system that produced the decision.

signature

Cryptographic proof that the artifact was generated by an authorized system.

timestamp

Time when the decision occurred.

version

Version of the protocol used.

Canonicalization

Before signing, the artifact must be serialized using canonical JSON.

Canonical serialization ensures that all verifiers compute the same hash.

Signing Process

Canonicalize the JSON artifact

Compute a hash

Sign the hash using the private key

Verification

Verification requires:

canonical serialization

public key

signature value
